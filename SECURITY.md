# Security Policy

## Purpose

This repository contains Google Dork patterns for **defensive security research only**.
This document outlines responsible use expectations and how to report concerns.

---

## ✅ Authorized Use

You may use these dorks for:
- Auditing systems you own or have **written authorization** to test
- Bug bounty programs where dorking is within the defined scope
- Academic or educational security research
- Defensive monitoring of your own organization's exposure

## ❌ Unauthorized Use

The following are explicitly prohibited:
- Running dorks against systems without permission
- Using results to access, exfiltrate, or damage data
- Targeting specific individuals, organizations, or infrastructure
- Any activity violating local, national, or international law

---

## 🐛 Reporting a Concern About This Repo

**If you find a dork that appears designed for targeted malicious use:**
Open a GitHub Issue with the label `misuse-report` and describe the concern.
We review all reports and will remove content that crosses ethical lines.

**If you find a dork in this repo actively being exploited:**
Please contact the maintainer directly via GitHub or open a private
security advisory using GitHub's built-in Security tab.

---

## 🔍 If You Find a Real Vulnerability Using These Dorks

1. **Do NOT exploit it** — accessing unauthorized systems is illegal
2. **Report it responsibly** to the affected organization via their
   security disclosure or bug bounty program
3. **Do not publish** details publicly before the organization has patched it
4. Consider documenting your methodology as an educational writeup

Helpful resources for responsible disclosure:
- [HackerOne Disclosure Guidelines](https://www.hackerone.com/disclosure-guidelines)
- [Bugcrowd Responsible Disclosure](https://www.bugcrowd.com/resource/what-is-responsible-disclosure/)
